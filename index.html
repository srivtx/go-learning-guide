Your First Go Program

Let's start with the classic "Hello, World!" program:

hello.go 
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}

Explanation:

  • package main - Every Go program starts with a package declaration
  • import "fmt" - Imports the fmt package for formatted I/O
  • func main() - The main function is the entry point of the program
  • fmt.Println() - Prints text to the console

Basic Go Syntax

Learn the fundamental syntax elements of Go programming.

Variables and Constants

variables.go 
package main

import "fmt"

func main() {
    // Variable declarations
    var name string = "Go"
    age := 25 // Short declaration
    
    // Constants
    const pi = 3.14159
    
    fmt.Println(name, age, pi)
}

Go supports both explicit and implicit variable declarations. Use 'var' for explicit typing or ':=' for type inference.

Functions

functions.go 
package main

import "fmt"

// Function with parameters and return type
func add(a, b int) int {
    return a + b
}

// Function returning multiple values
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("division by zero")
    }
    return a / b, nil
}

func main() {
    result := add(5, 3)
    fmt.Println("Sum:", result)
    
    quotient, err := divide(10, 2)
    if err != nil {
        fmt.Println("Error:", err)
    } else {
        fmt.Println("Quotient:", quotient)
    }
}

Functions in Go can return multiple values, commonly used for error handling. Function parameters and return types are explicitly declared.

Control Structures

control.go 
package main

import "fmt"

func main() {
    // If-else statement
    x := 10
    if x > 5 {
        fmt.Println("x is greater than 5")
    } else {
        fmt.Println("x is 5 or less")
    }
    
    // For loop
    for i := 0; i < 3; i++ {
        fmt.Printf("Count: %d\n", i)
    }
    
    // Range loop
    numbers := []int{1, 2, 3, 4, 5}
    for index, value := range numbers {
        fmt.Printf("Index: %d, Value: %d\n", index, value)
    }
    
    // Switch statement
    day := "Monday"
    switch day {
    case "Monday":
        fmt.Println("Start of the work week")
    case "Friday":
        fmt.Println("TGIF!")
    default:
        fmt.Println("Another day")
    }
}

Go has familiar control structures but with Go-specific syntax. The 'range' keyword is powerful for iterating over collections.

Structs and Methods

‚ñº
structs.go 
package main

import "fmt"

type Person struct {
    Name string
    Age  int
}

// Method with receiver
func (p Person) Greet() string {
    return fmt.Sprintf("Hello, I'm %s and I'm %d years old", p.Name, p.Age)
}

// Method with pointer receiver
func (p *Person) HaveBirthday() {
    p.Age++
}

func main() {
    person := Person{Name: "Alice", Age: 30}
    fmt.Println(person.Greet())
    
    person.HaveBirthday()
    fmt.Println(person.Greet())
}

Structs in Go are like classes in other languages. Methods can have value or pointer receivers.

Arrays and Slices

‚ñº
slices.go 
package main

import "fmt"

func main() {
    // Array (fixed size)
    var arr [5]int = [5]int{1, 2, 3, 4, 5}
    fmt.Println("Array:", arr)
    
    // Slice (dynamic)
    slice := []int{1, 2, 3}
    fmt.Println("Slice:", slice)
    
    // Append to slice
    slice = append(slice, 4, 5)
    fmt.Println("After append:", slice)
    
    // Slice operations
    subSlice := slice[1:4]
    fmt.Println("Sub-slice:", subSlice)
    
    // Make slice with capacity
    numbers := make([]int, 5, 10)
    fmt.Printf("Length: %d, Capacity: %d\n", len(numbers), cap(numbers))
}

Arrays have fixed size, slices are dynamic. Slices are more commonly used and provide powerful operations.

Error Handling

‚ñº
errors.go 
package main

import (
    "errors"
    "fmt"
)

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("cannot divide by zero")
    }
    return a / b, nil
}

func main() {
    // Handle error explicitly
    result, err := divide(10, 2)
    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    fmt.Println("Result:", result)
    
    // Handle division by zero
    result, err = divide(10, 0)
    if err != nil {
        fmt.Println("Error:", err)
    }
}

Go uses explicit error handling. Functions return an error as the last return value, which must be checked.

Goroutines (Concurrency)

‚ñº
goroutines.go 
package main

import (
    "fmt"
    "time"
)

func worker(id int, jobs <-chan int, results chan<- int) {
    for j := range jobs {
        fmt.Printf("Worker %d processing job %d\n", id, j)
        time.Sleep(time.Millisecond * 100)
        results <- j * 2
    }
}

func main() {
    jobs := make(chan int, 100)
    results := make(chan int, 100)
    
    // Start 3 workers
    for w := 1; w <= 3; w++ {
        go worker(w, jobs, results)
    }
    
    // Send 5 jobs
    for j := 1; j <= 5; j++ {
        jobs <- j
    }
    close(jobs)
    
    // Collect results
    for r := 1; r <= 5; r++ {
        <-results
    }
    
    fmt.Println("All jobs completed")
}

Goroutines are lightweight threads. Channels provide safe communication between goroutines.
